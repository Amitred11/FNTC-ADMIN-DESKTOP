App Documentation

A comprehensive guide to managing the FiBear Network system.

Understanding User Roles

This admin panel has three distinct user roles, each with specific permissions. The tabs and features you see are based on your assigned role.

Super Admin

  • Has full system oversight. Can manage users, subscriptions, billing, job orders, tickets, and system configuration.

Collector

  • Focused on financial tasks. Can view the dashboard and manage the Billing section to track payments and view invoices.

Field Agent

  • Responsible for on-site tasks. Can view their assigned Job Orders and handle related Support Tickets.

Dashboard Overview

The Dashboard provides a real-time snapshot of system health. Key metrics include total subscribers, pending payments, and open tickets. The statistics you see may be tailored to your role.

Core Workflow: Managing a New Subscriber

The "Action Inbox" (or Pending tab) is the central hub for pending tasks. This workflow is primarily managed by users with the Super Admin role.

Step 1: Approve Verification

  • An application in Pending Application status appears. Review the user's details and click "Approve" to create the initial bill and a job order.

Step 2: Installation

A field agent is assigned the job order. No action is required from the admin until the installation is complete.

Step 3: Activate Subscription

  • Once the field agent confirms installation is complete, find the user in the Pending Installation section and click "Activate" to start their billing cycle.

Ticket Management

This section is for formal support requests and is available to both Super Admins and Field Agents to coordinate on technical issues.

  • You can send one comprehensive reply per ticket. This reply can be edited or deleted.
  • Update the ticket's status (In Progress, Resolved) to keep the user informed.

Live Chat

Live Chat provides real-time assistance. An unread dot indicates new user messages. This feature is exclusive to Super Admins.

  • After the conversation, you must close the chat to archive it.

User Management

The "User Management" screen is a full directory of every user. This is an exclusive feature for the Super Admin role.

  • Search for users by name or email.
  • Tap any user to see their detailed profile, including all associated subscriptions, bills, and tickets.

Activity Log

The Activity Log provides a chronological timeline of all significant system events, useful for auditing. Access is limited to Super Admins.

System Configuration

This screen allows Super Admins to manage dynamic API keys for services like SendGrid (emails) and Xendit (payments).

Getting Help

If you encounter issues or have questions not covered in this guide, please use the "Contact Support" option in the Settings tab.
